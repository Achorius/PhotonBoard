PhotonBoard

Guide d'installation rapide

1. Installation

  1. Ouvrir Raspberry Pi Imager.
  2. Choisir l'OS → Use custom → sélectionner PhotonBoard-v0.1.0-<date>.img.gz.
  3. Cliquer sur l'engrenage pour personnaliser : hostname (ex. : AulaFlorence), Wi-Fi de la salle, utilisateur/mot de passe.
  4. Carte SD cible → flasher (16 Go minimum, la partition s'agrandit toute seule au premier boot).
  5. Carte SD dans le Pi 5 → brancher les écrans HDMI → alimenter → c'est prêt !

2. Accès

ServiceAdresse
PhotonBoard (3D & Stage)http://<hostname>.local:9091
Bitfocus Companionhttp://<hostname>.local:8000
API WebSocket (Companion, OSC…)ws://<hostname>.local:9090
SSHssh admin@<hostname>.local
Le hostname par défaut est PhotonBoard si vous n'en choisissez pas dans Pi Imager. Le fuseau horaire est Europe/Paris.

3. Identifiants par défaut

Utilisateur :admin
Mot de passe :admin

4. Charger un show depuis un ordinateur

  1. Ouvrir http://<hostname>.local:9091 dans Chrome ou Safari.
  2. Cliquer sur ↑ Upload Show dans la toolbar.
  3. Sélectionner un fichier .pbshow.
  4. Le show est transféré et reste sur le Pi (l'ordinateur peut être débranché).

5. Prérequis matériel

  • Raspberry Pi 5 (4 Go RAM minimum, 8 Go recommandé pour le dual-screen)
  • Carte SD 16 Go ou plus (tient sur n'importe quelle SD à partir de 16 Go)
  • Connexion Ethernet (recommandé) ou Wi-Fi configuré via Pi Imager
  • Optionnel : deux écrans tactiles HDMI wch.cn pour la fenêtre Stage

Configuration avancée

6. Mode Remote (Mac, tablette, téléphone)

Tout appareil du même réseau Wi-Fi peut piloter PhotonBoard depuis un navigateur. La synchronisation est bidirectionnelle : un déplacement de scène dans le navigateur apparaît immédiatement sur le Pi et inversement.

AppareilAffichage par défaut
Mac / PCInterface complète (3D, Timeline, Stage…)
TabletteInterface complète avec bouton Stage
TéléphoneSurface Stage plein écran (32 scènes + GM + Blackout)

7. Écrans tactiles

L'image associe automatiquement chaque dalle USB wch.cn à sa sortie HDMI via le chemin USB physique (stable après redémarrage).

  • Si gauche/droite sont inversés, éditer /etc/photonboard/touch-map.conf et passer SWAP=0 à SWAP=1, puis redémarrer.
  • Logs du mapping : /tmp/photonboard-touch-map.log

8. Bitfocus Companion

  • Companion est pré-installé et démarre automatiquement au boot.
  • Web UI : http://<hostname>.local:8000
  • Le module companion-module-photonboard est installé en mode developer — il apparaît dans la liste des connexions.
  • Configurer : cible localhost, port 9090.

9. Régénérer une image distribuable

Sur le Mac (Docker Desktop démarré) : bash scripts/pi-clone.sh --remote admin@<host>.local --password admin --output dist. L'image finale (~1.5 Go) apparaît dans dist/ et tient sur toute SD ≥ 16 Go.
